在Linux上使用 Github Pages 和 Hexo 搭建个人 blog

Posted on 2025-06-29 Edited on 2025-12-23 In 学习记录

前情提要

最近突发奇想，想要整理一下个人博客。作为一个程序猿还是需要有一点沉淀，也把我踩坑的历程记录下来吧。关于Github Pages和hexo的介绍我这里也就不赘述了，想了解的自行去看 Github Pages 和 Hexo 的文档吧。
我的个人开发机是Ubuntu系统，搭建hexo的环境全部是在Ubuntu上操作的，如果你也是用的Ubuntu或者其他内核的Linux，希望可以帮到你。

创建github仓库

正所谓免费的才是最香的，并且自己买服务器来搭建也并不轻松，所以就直接github的page功能来搞了。

首先你要有一个github的账号，如果没有那么你就自己先去注册一个，至于怎么注册，这里不再赘述。有账号以后，首先新建一个仓库，这里重点关注两个选项一个，名称 和 可见属性。其他按需填写即可，不理解的话，就不用管，直接点击Create repository。

  • 名称必须为xxx.github.io，xxx为你github账号的名字，我这里因为已经创建过了，所以报错已存在
  • 可见属性必须选择Pulic

这一步完成后，在仓库设置里，点击Pages页面，就可以看到我们的网页就已经生成好了，点击Visit site就可以直接跳转到我们的个人网页了。其余的配置，如果没有特殊需求就可以先按照我下面的方式来配置就好。

在github上配置ssh公钥，这个主要是方便我们拉代码和部署前端代码。在自己的电脑上找到公钥，一般是在目录 ~/.ssh/下，名称为xxx.pub，把内容复制出来，如果文件不存在，那么用下面的命令生成一个。

在github点击自己头像，按下图的方式添加就好。
